# 소스 코드 상세 설명

이 폴더는 'AI 도로 파손 분석기'의 핵심 로직과 GUI 어플리케이션 코드를 포함합니다. 프로젝트는 크게 세 가지 핵심 모듈(`utils`, `application`, `test`)로 구성되어, 역할과 책임이 명확하게 분리된 객체 지향적 설계를 따릅니다.

---

## 1. `utils_road_damage_analyzer.py` (핵심 기능 라이브러리)

* **목표:** 프로젝트 전반에서 사용되는 모든 핵심 '계산' 및 '시각화' 도구 함수들을 하나의 독립된 모듈로 캡슐화합니다. 이를 통해 코드의 재사용성을 극대화하고, 메인 어플리케이션 코드의 가독성을 높이는 것을 목표로 합니다.
* **주요 기능 상세:**

  * **`generate_depth_and_points(image_roi)`**:

    * 입력된 이미지 영역(ROI)을 받아, 흑백 변환을 통해 픽셀 밝기 값을 추출합니다.
    * 이 밝기 값을 Z 좌표로 사용하여, 시각화된 `Depth Map`과 `(X, Y, Z)` 형태의 `3D 포인트 클라우드` Numpy 배열을 생성하여 반환합니다.

  * **`analyze_severity_from_points(points_3d)`**:

    * `3D 포인트 클라우드` 데이터를 입력받아, Z값의 분포(평균, 표준편차)를 통계적으로 분석합니다.
    * '평균 깊이(어두움)'와 '표면 거칠기(밝기 표준편차)'라는 두 가지 지표를 기반으로, 미리 정의된 임계값에 따라 파손의 심각도를 '높음/중간/낮음' 3단계로 분류하고, 시각화를 위한 색상 정보를 함께 반환합니다.

  * **`create_3d_plot_image(points_3d, title)`**:

    * `Matplotlib`을 사용하여 입력된 `3D 포인트 클라우드` 데이터의 3D 산점도(Scatter Plot)를 생성합니다.
    * 렌더링된 그래프를 직접 화면에 띄우는 대신, 메모리 버퍼(`io.BytesIO`)를 이용해 PNG 이미지 데이터로 변환한 뒤, `cv2.imdecode`를 통해 OpenCV가 처리할 수 있는 Numpy 이미지 배열로 반환하는 고도화된 기법을 사용합니다.

  * **`draw_korean_text(...)` & `add_panel_title(...)`**:

    * `Pillow` 라이브러리를 활용하여, OpenCV의 한글 깨짐 문제를 해결하고 이미지 위에 깔끔한 한글 텍스트와 제목을 그리는 유틸리티 함수입니다.

## 2. `run_road_damage_analyzer.py` (GUI 어플리케이션)

* **목표:** 사용자가 직관적으로 상호작용할 수 있는 GUI(Graphical User Interface)를 제공하여, 복잡한 AI 분석 과정을 쉽고 편리하게 실행할 수 있도록 합니다.
* **주요 기능 상세:**

  * **UI 구성 (`__init__`)**:

    * `Tkinter` 라이브러리를 사용하여 어플리케이션의 메인 윈도우를 생성합니다.
    * `ttk.Combobox`로 여러 AI 모델 중 하나를 선택할 수 있게 하고, `ttk.Button`으로 파일 선택 및 분석/저장 기능을 제공하는 등, 사용자 친화적인 인터페이스를 구성합니다.
    * `grid` 레이아웃 매니저와 `Canvas` 위젯을 사용하여, 창 크기가 변해도 내용물이 깨지지 않는 반응형 UI를 구현했습니다.

  * **비동기 처리 (`threading`)**:

    * AI 모델 로드 및 이미지 분석과 같이 시간이 오래 걸리는 핵심 작업은 **별도의 백그라운드 스레드(`threading.Thread`)**에서 실행됩니다.
    * 이를 통해 분석이 진행되는 동안에도 GUI가 멈추거나 응답 없 상태가 되지 않는, 매우 부드러운 사용자 경험(UX)을 제공합니다.

  * **스레드 안전성 (`self.root.after`)**:

    * 백그라운드 스레드는 직접 GUI 위젯(로그 창, 진행률 바, 이미지 캔버스)을 업데이트하지 않습니다.
    * 대신, `self.root.after(0, ...)` 메소드를 사용하여 모든 GUI 업데이트 요청을 **메인 스레드의 작업 큐에 안전하게 전달**합니다. 이는 GUI 프로그래밍에서 스레드 충돌을 방지하는 매우 중요한 표준 기법입니다.

  * **통합 대시보드 생성 (`update_gui_with_results`)**:

    * AI 분석이 완료되면, 백그라운드 스레드로부터 결과 데이터를 전달받습니다.
    * `utils.py`의 도구 함수들을 호출하여 '원본 이미지', 'AI 분석 결과', '주요 영역 3D 뷰', '상세 3D 분석 리포트' 4개의 패널을 각각 생성합니다.
    * `cv2.hconcat`과 `cv2.vconcat`을 사용하여 4개의 패널을 하나의 통합된 대시보드 이미지로 완벽하게 합성한 뒤, 최종적으로 화면에 표시합니다.

## 3\. `test_road_damage_analyzer.py` (Unit Test)

* **목표:** `utils.py`에 정의된 모든 핵심 함수들의 안정성과 신뢰성을 `pytest`를 통해 객관적으로 검증하여, 소프트웨어를 보증합니다.
* **주요 기능 상세:**

  * **모킹 (`unittest.mock.patch`)**: `Pillow`의 폰트 로딩 기능처럼, 파일 시스템에 접근하거나 외부 의존성이 있는 부분을 **'가짜 객체(Mock)'**로 대체했습니다. 이를 통해 네트워크나 파일 유무에 상관없이, 오직 코드의 논리적 정확성만을 독립적으로 테스트하는 순수한 유닛 테스트 환경을 구축했습니다.
  * **파라미터화 테스트 (`@pytest.mark.parametrize`)**: `analyze_severity_from_points` 함수의 심각도 분류 로직을 검증하기 위해, 다양한 경계값과 시나리오를 하나의 테스트 함수로 묶어 효율적으로 테스트했습니다.
  * **예외 처리 검증 (`pytest.raises`)**: 정상적인 입력뿐만 아니라, `None`, 빈 이미지, 잘못된 데이터 타입 등 **다양한 예외 상황(Edge Case)**에 대해, 각 함수가 의도한 대로 정확한 에러를 발생시키는지 검증하여 코드의 견고함을 확보했습니다.
