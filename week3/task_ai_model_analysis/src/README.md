## 스크립트 상세 설명

### 1. `1_run_training.py`
- **목표:** 모든 성능 비교의 기준점이 될 기본(Baseline) 모델을 생성합니다.
- **주요 로직:**
  1. `YOLO("yolov8n.pt")`를 통해 사전 훈련된 YOLOv8 Nano 모델을 로드합니다.
  2. `model.train()` 함수를 호출하여, `datasets/data.yaml` 설정 파일을 기반으로 10 Epochs 동안 모델을 훈련(Fine-tuning)시킵니다.
  3. `project`와 `name` 인자를 사용하여, 훈련 결과가 다른 실험과 섞이지 않도록 `runs/training/yolov8n` 폴더에 저장되도록 경로를 지정합니다.
  4. 훈련 완료 후, `model.val()` 함수를 통해 검증 데이터셋에 대한 모델의 최종 성능(mAP 등)을 측정하고 터미널에 출력합니다.
- **실행 결과:** `runs/training/yolov8n/` 폴더에 훈련 과정 그래프, 성능 지표(`results.csv`), 그리고 가장 성능이 좋았던 모델(`weights/best.pt`)이 저장됩니다.

### 2. `2_run_training_augmented.py`
- **목표:** 데이터 증강(Data Augmentation) 기법이 모델의 일반화 성능에 미치는 영향을 확인합니다.
- **주요 로직:**
  1. 기준 모델과 동일한 `YOLOv8n` 모델과 `10 Epochs` 조건을 사용합니다.
  2. `model.train()` 함수에 `augment=True` 옵션을 추가합니다. 이 옵션은 훈련 시 이미지의 회전, 밝기, 색조 등을 무작위로 변경하여, 모델이 더 다양한 상황을 학습하도록 유도합니다.
  3. `name='yolov8n_augmented'`를 지정하여, 실험 결과가 기준 모델과 겹치지 않도록 별도의 폴더에 저장합니다.
- **실행 결과:** `runs/tuning/yolov8n_augmented/` 폴더에 데이터 증강을 적용한 모델의 훈련 결과가 저장됩니다.

### 3. `3_run_training_50_epoch.py`
- **목표:** 충분한 학습 시간(Epochs) 확보가 모델 성능에 미치는 영향을 확인합니다.
- **주요 로직:**
  1. 기준 모델과 동일한 `YOLOv8n` 모델을 사용합니다.
  2. `model.train()` 함수에서 학습 반복 횟수를 `epochs=50`으로 5배 늘립니다. 이를 통해 모델이 데이터의 패턴을 더 깊이 있고 안정적으로 학습하도록 합니다.
  3. `name='yolov8n_50_epoch'`를 지정하여, 실험 결과가 별도의 폴더에 저장되도록 합니다.
- **실행 결과:** `runs/tuning/yolov8n_50_epoch/` 폴더에 50 Epochs 동안 훈련한 모델의 결과가 저장됩니다.

### 4. `4_run_training_yolov8s.py`
- **목표:** 더 크고 복잡한 모델이 더 높은 성능을 보이는지 확인합니다.
- **주요 로직:**
  1. `YOLO("yolov8s.pt")`를 사용하여, 기준 모델(`n`)보다 파라미터 수가 더 많은 'Small' 버전의 모델을 로드합니다.
  2. `model.train()` 함수를 통해 10 Epochs 동안 동일한 데이터셋으로 훈련을 진행합니다.
  3. `name='yolov8s'`를 지정하여, 실험 결과가 별도의 폴더에 저장되도록 합니다.
- **실행 결과:** `runs/tuning/yolov8s/` 폴더에 YOLOv8s 모델의 훈련 결과가 저장됩니다.

### 5. `5_run_detection_all_models.py`
- **목표:** 4개의 훈련된 모델들이 새로운 실사 이미지를 어떻게 예측하는지 시각적으로 비교하고, 각 모델의 강점과 약점을 정성적으로 분석합니다.
- **주요 로직:**
  1. 4개 실험에서 생성된 `best.pt` 모델의 경로를 딕셔너리 형태로 정의합니다.
  2. `for` 루프를 사용하여 각 모델을 순서대로 불러온 뒤, `input_data` 폴더의 동일한 테스트 이미지에 대해 객체 탐지를 수행합니다.
  3. `results[0].plot()` 함수로 탐지 결과를 이미지 위에 그리고, `cv2.imwrite()`를 사용하여 모델 이름이 포함된 별도의 파일로 `outputs` 폴더에 저장합니다.
- **실행 결과:** `outputs/detection_results/` 폴더에 4개의 모델이 각각 동일한 이미지를 예측한 결과 이미지 파일 4장이 생성됩니다.

### 6. `6_analyze_training_results.py`
- **목표:** 4개 모델의 훈련 과정에서 기록된 정량적 성능 지표(mAP)를 하나의 그래프로 시각화하여, 어떤 튜닝 전략이 가장 효과적이었는지 객관적으로 비교 분석합니다.
- **주요 로직:**
  1. 4개 실험 결과 폴더에 있는 `results.csv` 파일의 경로를 딕셔너리 형태로 정의합니다.
  2. `pandas` 라이브러리를 사용하여 각 csv 파일에서 epoch별 `mAP` 점수 데이터를 추출합니다. 이때, 열 이름의 보이지 않는 공백을 `str.strip()`으로 제거하여 데이터 로드 오류를 방지합니다.
  3. `matplotlib` 라이브러리를 사용하여, 4개 모델의 성능 변화를 하나의 꺾은선 그래프로 시각화합니다. 각 그래프는 다른 색상과 스타일로 구분하여 가독성을 높입니다.
- **실행 결과:** 4개 실험의 성능 변화를 한눈에 비교할 수 있는 종합 그래프가 화면에 출력됩니다.
